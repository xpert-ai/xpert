# 三、核心模块设计

## 1️⃣ Event Schema（统一事件协议）

👉 这是最重要的，不统一你后面全乱

```ts
export interface XpertEvent<T = any> {
  id: string              // event id (uuid)
  type: string            // event type
  scope: {
    projectId?: string
    taskId?: string
    agentId?: string
  }
  source: {
    type: 'agent' | 'workflow' | 'system' | 'tool'
    id: string
  }
  payload: T
  meta?: {
    userId?: string
    traceId?: string
  }
  timestamp: number
}
```

## 3️⃣ Event SDK（统一发事件入口）

👉 所有 Worker / Agent 都必须通过它发事件

```ts
@Injectable()
export class EventPublisher {
  constructor(private redis: Redis) {}

  async publish(event: XpertEvent) {
    await this.redis.xadd(
      'xpert:events',
      '*',
      'data',
      JSON.stringify(event)
    )
  }
}
```

---

## 4️⃣ Event Bus（推荐 Redis Stream）

为什么不用 Pub/Sub？

| 对比   | Pub/Sub | Redis Stream |
| ---- | ------- | ------------ |
| 持久化  | ❌       | ✅            |
| 可回放  | ❌       | ✅            |
| 消费确认 | ❌       | ✅            |

👉 **推荐：Redis Stream**

---

## 5️⃣ Event Gateway（事件网关）

👉 核心职责：

* 从 Redis Stream 读取事件
* 按 projectId / taskId 分发
* 推送给前端

```ts
@Injectable()
export class EventGateway {
  private projectStreams = new Map<string, Subject<XpertEvent>>()

  async start() {
    while (true) {
      const events = await this.readFromStream()

      for (const event of events) {
        const projectId = event.scope.projectId
        this.projectStreams.get(projectId)?.next(event)
      }
    }
  }

  getProjectStream(projectId: string) {
    if (!this.projectStreams.has(projectId)) {
      this.projectStreams.set(projectId, new Subject())
    }
    return this.projectStreams.get(projectId)!.asObservable()
  }
}
```

---

## 6️⃣ SSE 接口

```ts
@Controller('events')
export class EventController {
  constructor(private gateway: EventGateway) {}

  @Sse('projects/:projectId')
  stream(@Param('projectId') projectId: string) {
    return this.gateway.getProjectStream(projectId).pipe(
      map(e => ({ data: e }))
    )
  }
}
```

# 四、前端设计（Angular）

## 1️⃣ 事件驱动 Store

```ts
class TaskStore {
  tasks = signal<Map<string, Task>>(new Map())

  applyEvent(event: XpertEvent) {
    switch (event.type) {
      case 'task.status_changed':
        this.patchTask(event.scope.taskId, {
          status: event.payload.status
        })
        break

      case 'task.progress':
        this.patchTask(event.scope.taskId, {
          progress: event.payload.percent
        })
        break
    }
  }
}
```

👉 避免 replay 太慢
