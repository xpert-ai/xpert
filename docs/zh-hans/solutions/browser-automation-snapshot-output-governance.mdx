# 1. 产品定位

Browser Automation 的页面快照用于让智能体理解当前网页状态，并继续完成点击、填写、滚动、等待等操作。

本次调整后，页面快照的产品形态分为两类：

- 普通页面快照：保持完整输出。
- 大页面快照：自动精简后输出。

用户、调试人员和日志查看者看到的仍然是页面快照结果，但当页面内容过多时，系统会优先保留后续操作必需的信息，减少无关的大段页面细节。

---

# 2. 调整前后的变化

调整前：

- 页面快照会尽量返回完整页面结构。
- 复杂网页可能包含大量元素、布局、文本和内部定位信息。
- 日志中可能出现很长的快照内容。
- 对话执行中可能因为快照过大导致等待时间变长或回复卡住。

调整后：

- 小快照不变，仍展示完整页面信息。
- 大快照会变成精简版。
- 精简版保留页面标题、地址、视口、页面状态和可操作元素。
- 精简版移除或截断冗长的内部细节。
- 日志和预览中展示的是精简后的快照结果。

---

# 3. 用户看到的快照形态

## 3.1 普通快照

普通快照用于页面元素较少、内容较短的场景。

用户看到的内容包括：

- 当前页面地址
- 当前页面标题
- 页面尺寸和滚动位置
- 页面状态
- 可操作元素列表
- 每个元素的名称、类型和定位信息
- 可用的自动化能力

这类快照不会被精简。

## 3.2 精简快照

当页面很复杂、元素很多或快照内容很长时，系统输出精简快照。

用户看到的内容包括：

- 当前页面地址
- 当前页面标题
- 页面尺寸和滚动位置
- 页面状态
- 最近的网络或控制台状态
- 最多保留的一组关键可操作元素
- 每个元素的核心定位信息
- 快照已被精简的提示信息

用户不会再看到大量内部命中路径、完整层级路径、长文本堆栈和重复性页面细节。

---

# 4. 精简快照保留的页面信息

精简快照会保留页面级信息。

| 信息       | 用途                                 |
| ---------- | ------------------------------------ |
| 页面地址   | 判断当前是否在目标页面               |
| 页面标题   | 辅助识别页面上下文                   |
| 视口信息   | 判断页面可见区域大小                 |
| 滚动位置   | 判断当前浏览位置                     |
| 页面状态   | 判断页面是否加载完成、是否可继续操作 |
| 自动化能力 | 判断当前页面支持哪些操作方式         |
| 网络状态   | 辅助判断页面是否仍在请求数据         |
| 控制台错误 | 辅助判断页面是否出现运行错误         |
| 页面错误   | 辅助判断是否需要中断或重试           |

如果原始快照里没有某项信息，精简快照不会额外生成该信息。

---

# 5. 精简快照保留的元素信息

精简快照会保留一组关键可操作元素。

每个元素优先保留：

| 信息       | 用途                                   |
| ---------- | -------------------------------------- |
| 元素引用   | 后续点击、填写等操作优先使用           |
| 无障碍引用 | 辅助通过无障碍树定位元素               |
| 角色       | 判断元素是按钮、输入框、链接还是复选框 |
| 名称       | 识别元素对应的用户可见含义             |
| 标签类型   | 辅助判断元素类型                       |
| 可见文本   | 辅助确认元素内容                       |
| 选择器     | 元素引用不可用时的备用定位方式         |
| 当前值     | 输入框、选择框等控件的当前内容         |
| 占位文本   | 辅助识别空输入框                       |
| 是否启用   | 判断元素是否可以操作                   |
| 是否可见   | 判断元素是否在页面中可见               |
| 是否可操作 | 判断是否可以直接点击或填写             |
| 是否选中   | 判断复选框、单选框等状态               |
| 元素区域   | 支持基于位置的操作                     |
| 元素中心点 | 支持坐标点击等兜底操作                 |
| 附近文本   | 辅助理解元素周围上下文                 |

这些信息足够支持智能体继续执行常见页面操作。

---

# 6. 会被移除或截断的内容

精简快照会减少不适合长期进入对话和日志的大段内容。

常见被移除或截断的内容：

- 很长的页面文本
- 重复的附近文本
- 大量不可操作元素
- 完整 DOM 路径
- 内部命中路径
- 过长的布局细节
- 大数组字段
- 深层嵌套对象
- 对后续点击、填写没有直接帮助的调试字段

被截断的文本会带有截断提示，例如“已截断若干字符”或“已截断若干项”。

---

# 7. 精简提示信息

当快照被精简后，结果中会出现精简提示信息。

提示信息用于说明：

- 当前快照已经被精简
- 原始快照大约有多长
- 原始元素数量是多少
- 精简后保留了多少元素

该提示主要服务于日志查看和问题排查。

它不代表页面操作失败，也不代表智能体不能继续工作。

---

# 8. 日志和预览中的表现

在对话日志和预览面板中：

- 小快照展示完整结果。
- 大快照展示精简结果。
- 精简结果中可以看到“已精简”的提示信息。
- 过长字段不会再完整展开。
- 大量重复元素不会继续全部展示。

这会让日志更容易打开、浏览和定位问题。

对于使用者来说，最明显的变化是：

- 日志更短。
- 页面响应更稳定。
- 大页面任务不容易因为快照过大卡住。
- 仍然可以看到智能体下一步操作所需的关键页面信息。

---

# 9. 对智能体行为的影响

精简快照不会改变智能体可用的页面操作能力。

智能体仍然可以继续使用：

- 元素引用点击
- 无障碍引用点击
- 选择器定位
- 文本和角色定位
- 输入框填写
- 滚动
- 坐标兜底操作

可能发生变化的是：当页面特别复杂时，智能体看到的页面细节会减少。

因此后续操作会更依赖：

- 关键可操作元素
- 元素名称
- 元素角色
- 附近文本
- 坐标和区域信息

---

# 10. 产品验收标准

普通页面：

- 快照内容保持完整。
- 日志中不出现精简提示。
- 智能体正常点击、填写、滚动。

复杂页面：

- 快照内容会被精简。
- 日志中出现精简提示。
- 页面标题、地址、视口和页面状态仍可见。
- 可操作元素仍保留关键定位信息。
- 冗长内部字段不再大量展示。
- 对话不会因为快照过大长时间卡住。

日志体验：

- 大快照日志可以正常打开。
- 预览面板不会展示过长的重复内容。
- 排查人员能看到原始元素数量和保留元素数量。

---

# 11. 不在本次范围内

本次调整不包含：

- 改变 Browser Automation 的工具列表
- 改变页面采集方式
- 改变点击、填写、滚动的交互规则
- 改变截图能力
- 为用户提供手动切换完整快照和精简快照的入口
- 调整前端日志页面的视觉样式

这些可以作为后续产品能力单独设计。
